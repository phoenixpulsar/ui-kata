About

This application allows users to securely encrypt files using a password before transmitting or storing them.

Steps

  • File Upload

    The user uploads a file through the application interface. The application reads and stores the file's details, such as MIME type, filename, and file extension.

  • Password Input

    The user is prompted to enter and confirm a password. This password will be used to encrypt the file.

  • Encryption Process

    • Convert Data and Password to Bytes

      The file data is converted into a sequence of bytes. The password is also converted into bytes using a text encoder.

    • Import Key

      The byte representation of the password is imported as a key. This imported key will be used later to derive the actual encryption key.

    • Generate Salt

      A random salt of 32 bytes (256 bits) is generated. The salt ensures that even if the same password is used, the encryption will be unique.

    • Derive Encryption Key

      The password key and salt are used to derive an encryption key through a process called PBKDF2. This process involves hashing the password key with the salt 250,000 times using SHA-256. The derived key is used for AES-GCM encryption with a length of 256 bits.

    • Generate Initialization Vector (IV)

      A 12-byte (96-bit) initialization vector (IV) is generated. The IV ensures that the same plaintext encrypted with the same key will produce different ciphertexts.

    • Encrypt Data

      The file data is encrypted using the derived key and IV. The result is an encrypted content buffer.

    • Package Encrypted Data

      The salt, IV, and encrypted data are combined into one package. This package is converted into a base64 string, which is suitable for transmission.

  • Output

    The final output is a base64 encoded string of the encrypted package. This string can be securely stored or transmitted as needed.
